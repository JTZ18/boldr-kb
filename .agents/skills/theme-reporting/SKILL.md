# Theme Reporting

Use this skill for weekly theme artifacts.

## Steps

1. Read the theme payload and source ticket IDs.
2. Write a dated report under `briefs/themes/` or update a task-specified path.
3. Keep examples anonymized.
4. Link relevant KB entries if a theme maps to known facts.
5. Run validation.

## Rules

- Do not create KB facts from theme frequency alone.
- Use theme reports to inform future gaps, personas, and briefs.
- Include date window and input count in the report.
